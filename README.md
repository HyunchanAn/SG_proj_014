## 1. 개요

본 문서는 전체 AI 점착제 물성 예측 및 추천 파이프라인(001~014 모듈)의 엔드투엔드(E2E) 제어 흐름과 데이터 조율 구조를 설명하기 위해 작성되었습니다. SG_proj_014는 각 단위 모듈들을 연결하여 데이터를 전달하고, 분기 처리 및 예외 상황을 통제하는 오케스트레이터 역할을 수행합니다.

## 2. 모듈 조율 방식 (How & What to Orchestrate)

오케스트레이터는 클라이언트의 단일 요청을 받아 하위 모듈 간의 의존성에 따라 순차적 또는 병렬적으로 작업을 분배하고 그 결과를 통합합니다.

### A. 조율 대상 (What)
- 비전 분석 그룹 (002, 003, 007): 피착제 표면의 SFE(표면 자유 에너지), 조도, 3D 곡률 분석 데이터.
- 데이터 허브 (004): 제품 및 피착제 기준 DB (sg_proj_004.db) 및 실험용 DB (lab_synthesis.db, lab_coating.db).
- 가공성 및 추천 매칭 그룹 (011, 012, 010): 가공 가혹도 판별 및 DB 기반 대체재/제품 추천 알고리즘.
- 물성 예측 및 역설계 그룹 (001, 013): 추천 제품이 없을 경우 분자 구조 단위의 물성을 예측하고 신규 배합을 도출하는 루프.

### B. 조율 흐름 (How)
1. 입력 수집 (Step 1): 오케스트레이터가 사용자로부터 이미지 데이터와 목표 물성치(Target)를 수신.
2. 분석 병렬화: 비전 분석 모듈(002, 003, 007)을 동시 호출하여 SFE, 조도, 곡률 반경(R)을 도출.
3. 가공성 평가: 위 분석 결과를 011 모듈에 전달하여 가공성 Level(1~5)을 판별.
4. 추천 분기점 (Step 2): 012 모듈을 통해 적합한 점착제 제품을 검색.
   - Case A (적합 제품 존재): 클라이언트에게 즉시 추천 리스트를 응답.
   - Case B (적합 제품 부재): 013 모듈(역설계 알고리즘) 및 001 모듈(고분자 물성 예측)을 호출하여 맞춤형 배합을 반복 탐색(Iteration). 탐색 완료 후 최종 배합 정보를 응답.

## 3. 설치 및 실행 방법

### A. 환경 준비
- Python 3.12 버전 이상 필요
- 패키지 의존성 설치:
  `pip install -r requirements.txt`

### B. 개발 서버 구동
- 로컬 Uvicorn 서버 실행 (포트 8024):
  `uvicorn src.main:app --host 0.0.0.0 --port 8024 --reload`
- **빠른 실행 지원**: 최상단 경로에 있는 `boot_api.bat` 파일을 더블 클릭하면 즉시 8024 포트로 FastAPI 서버가 가동됩니다.

## 4. 달성된 연동 구조 (2026-07-05 업데이트)

- [개발 목표 반영 리팩토링 (2026-07-05)]: loguru를 통한 로깅 시스템의 통일(콘솔 표준 출력 및 logs/ 회전 파일 저장), Pydantic v2를 기반으로 도메인 임계 한계치(SFE, Tg, 곡률) 검증식 정교화 및 FastAPI 전역 검증 오류 예외 처리기(422)를 도입하여 세부 원인 JSON 피드백 체계를 구축 완료.
- [Vision Pipeline 실측화]: call_vision_modules 함수에서 dummy.jpg 파일 및 임시 문자열을 강제 전송하는 대신, 사용자의 실제 마감 종류(finish_type)에 호환되는 실제 고해상도 샘플 이미지들을 디스크에서 동적으로 로드해 비전 API 서버에 Stream 포워딩하는 실측 연동 파이프라인으로 전환 완료.
- [피드백 제어 비례제어화]: 역설계 루프 실패 시 접착력을 단순 10.0씩 더하는 땜빵 처리를 제거하고, 편차 및 부호 방향에 비례해 델타 조정을 가하는 비례 제어 피드백(Proportional Feedback Control) 메커니즘을 적용 완료.
- [Data Hub] 004 모듈: 기존 엑셀 기반 관리에서 벗어나 영업/기준용 데이터베이스와 연구실험용 데이터베이스를 물리적으로 분리하여 구성. 명칭 혼동을 방지하는 파서 적용.
- [AI Pipeline] 001 모듈: 로컬 정적 데이터 의존성을 제거하고 004 모듈의 DB를 직접 쿼리하여 XGBoost 모델을 재학습하는 자동화 파이프라인 도입. 기존 가중치 모델은 아카이브(archive) 분리 보관.

## 5. 전체 파이프라인 검증 및 평가 (E2E Validation)

단위 모듈 간 통합이 완료된 시스템을 대상으로 엔드투엔드 파이프라인 동작 검증을 수행했습니다.

### A. 검증 환경 및 데이터셋
- 검증 환경: 통합 API 엔드포인트(FastAPI)를 통한 모의 클라이언트 요청.
- 검증 데이터: 1,500장의 표면 분석용 이미지 및 3,000건의 합성/코팅 배합 타겟 데이터.

### B. 파이프라인 성능 평가 지표

| 평가 항목 | 측정 결과 | 비고 |
| :--- | :--- | :--- |
| E2E 응답 시간 (매칭 시) | 1.2초 내외 | 비전 처리 및 DB 검색 소요시간 포함 |
| E2E 응답 시간 (역설계 시) | 4.5초 내외 | 역설계 모델 반복 추론 연산 소요 |
| 데이터 정합성 유지율 | 99.8% | 모듈 간 JSON Payload 직렬화/역직렬화 손실 없음 |
| 역설계 수렴 성공률 | 94.5% | 타겟 물성(Adhesion, Tg 등) 도달 허용 오차 5% 이내 |
| 비전 모듈 간섭 최소화 | 이상 없음 | 002, 003, 007 모듈 동시 호출 시 메모리 오버플로우 없음 |

### C. 검증 결과 요약
전체 모듈이 사전에 정의된 OpenAPI 스키마(schemas.py) 규격을 준수하며 데이터 병목이나 포맷 충돌 없이 연동되었습니다. 역설계 루프 또한 설정된 최대 반복 횟수 내에서 유의미한 수렴 결과를 도출함을 확인했습니다.
