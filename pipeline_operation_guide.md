# SG 플랫폼 파이프라인 가동 및 감시 가이드

본 문서는 SG 플랫폼 통합 파이프라인을 가동하고 각 독립 모듈 간의 연결 상태를 감시하는 방법을 설명합니다. 전체 파이프라인은 데이터베이스, 가공성 평가, 매칭, 역설계 및 중앙 오케스트레이터로 구성되어 상호 유기적으로 연동됩니다.

## 1. 파이프라인 구성 모듈 및 기본 포트 정보

각 독립 모듈은 로컬 환경에서 지정된 포트를 사용하여 API 서비스를 제공합니다.

* SG_proj_004 (데이터베이스 및 쿼리): http://localhost:8004
* SG_proj_011 (가공성 판별): http://localhost:8011
* SG_proj_012 (제품 매칭): http://localhost:8012
* SG_proj_013 (역설계 검증): http://localhost:8013
* SG_proj_014 (중앙 오케스트레이터): http://localhost:8014

## 2. 모듈 가동 가이드

원활한 통합 테스트 및 파이프라인 구동을 위해 아래 순서대로 각 모듈의 서버를 구동합니다.

### 1단계: SG_proj_004 데이터베이스 백엔드 가동
* 전제 조건: Docker Desktop이 켜져 있어야 하며, PostgreSQL 및 Qdrant 컨테이너가 정상 실행 상태여야 합니다.
  * 실행 확인 명령어: `docker ps`
  * 실행 중이어야 할 컨테이너: sg_proj_004_db, sg_proj_004_qdrant
* FastAPI 서버 실행 방법:
  1. SG_proj_004 디렉토리로 이동합니다.
  2. 가상환경 활성 상태에서 uvicorn 서버를 실행합니다.
     * 실행 명령어: `.venv\Scripts\uvicorn src.main:app --port 8004`

### 2단계: 개별 물리 모듈 (011, 012, 013) 가동
* 각 프로젝트 디렉토리로 이동한 뒤, 해당 가상환경 또는 도커 컨테이너를 가동합니다.
* 011 모듈 가동 (포트 8011): `.venv\Scripts\uvicorn src.main:app --port 8011` (예시)
* 012 모듈 가동 (포트 8012): `.venv\Scripts\uvicorn src.main:app --port 8012` (예시)
* 013 모듈 가동 (포트 8013): `.venv\Scripts\uvicorn src.main:app --port 8013` (예시)
* 실제 통합 연동 시에는 개별 모듈 컨테이너가 항상 가동되어 있어야 연동 테스트가 통과합니다.

### 3단계: SG_proj_014 오케스트레이터 가동
* 오케스트레이터는 파이프라인의 중심점으로서 각 모듈을 중계합니다.
* 가동 명령어:
  1. SG_proj_014 디렉토리로 이동합니다.
  2. 가상환경 활성 상태에서 서버를 실행합니다.
     * 실행 명령어: `.venv\Scripts\uvicorn src.main:app --port 8014 --reload`

## 3. 파이프라인 감시 및 연결성 검증 (Health Check)

시스템 가동 상태를 감시하고 모듈 간 실시간 연결 상태를 진단하기 위해 통합 E2E 테스트 스크립트가 구현되어 있습니다.

* 테스트 파일 위치: SG_proj_014/cross_module_tests/test_e2e_pipeline.py
* 실행 방법:
  1. SG_proj_014 디렉토리에서 아래 명령어로 테스트를 구동합니다.
     * 실행 명령어: `pytest cross_module_tests/test_e2e_pipeline.py -s`
* 검증 시나리오:
  1. test_module_health_checks:
     * 5개 모듈(004, 011, 012, 013, 014)의 API 서버가 정상 기동 중인지 순차적으로 검증합니다.
     * Mock 없이 실제 HTTP 호출을 수행하며, 비가동 중인 모듈 발견 시 명확한 연결 에러(FAIL)를 보고합니다.
  2. test_full_pipeline_e2e:
     * 004 데이터베이스 모듈에서 실제 피착재 사양을 조회합니다.
     * 조회된 피착재 데이터를 014 오케스트레이터의 입력 페이로드로 가공하여 파이프라인 워크플로우를 가동합니다.
     * 각 모듈 연동의 비즈니스 논리(매칭 성공 또는 역설계 가동) 및 반환 데이터 구조를 최종 검증합니다.

## 4. 디버깅 및 트러블슈팅

* 연결 실패 (Connection Error):
  * 특정 포트가 닫혀 있는 경우 에러 리포트에 기재된 포트 번호를 확인하고, 해당 모듈의 uvicorn 서버 또는 컨테이너가 구동 중인지 점검하십시오.
* 데이터베이스 조회 실패:
  * 004 모듈의 PostgreSQL 컨테이너에 데이터가 정상 적재되지 않았거나 테이블이 유실되었을 수 있습니다. 004 모듈 내 src/etl.py 파이프라인을 실행하여 엑셀 원본 데이터를 다시 데이터베이스에 적재하십시오.
